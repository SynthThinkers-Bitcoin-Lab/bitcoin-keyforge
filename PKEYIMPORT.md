# üîë Private Key Import Guide

This guide explains how to import private keys generated by `bitcoin-keyforge` into various Bitcoin wallets.

## ‚ö†Ô∏è Security Warning

**CRITICAL: Never import private keys on compromised or online systems for production use!**

- Use only on air-gapped, secure machines for significant amounts
- Test with small amounts first
- Verify addresses match before sending funds
- Keep backup copies of your private keys secure
- Consider hardware wallets for long-term storage

## üìã Supported Import Formats

`bitcoin-keyforge` generates private keys in these formats:

- **Hex Format**: `da84c6bb3701b2c...`
- **WIF Format**: `L2yeAsCPNn6KH8c...` (mainnet)
- **WIF Format**: `cSC88ZBoGVi4pwe...` (testnet)

Most wallets accept **WIF format** for imports.

## üíº Desktop Wallets

### Bitcoin Core (Official Client)

**Method 1: Console Import**
```bash
# Open Bitcoin Core console (Help ‚Üí Debug Window ‚Üí Console)
importprivkey "L2yeAsCPNn6KH8c..."

# With label (optional)
importprivkey "L2yeAsCPNn6KH8c..." "my-imported-key"

# Rescan blockchain (may take time)
rescanblockchain
```

**Method 2: RPC Command**
```bash
bitcoin-cli importprivkey "L2yeAsCPNn6KH8c..."
```

### Electrum

1. **Create New Wallet**:
   - File ‚Üí New/Restore
   - Choose "Import Bitcoin addresses or private keys"

2. **Import Private Key**:
   ```
   L2yeAsCPNn6KH8c...
   ```

3. **Multiple Keys** (one per line):
   ```
   L2yeAsCPNn6KH8c...
   KyZ7VriWmDvFgnb...
   ```

### Sparrow Wallet

1. **File ‚Üí Import Wallet**
2. **Select "Private Key"**
3. **Paste WIF**:
   ```
   L2yeAsCPNn6KH8c...
   ```
4. **Choose address type** (P2PKH, P2WPKH, etc.)

### Wasabi Wallet

1. **Wallet ‚Üí Load Wallet ‚Üí Import**
2. **Select "Private Keys"**
3. **Enter WIF format**
4. **Choose derivation path**

## üîß Advanced Import Methods

### Using Bitcoin CLI

```bash
# Basic import
bitcoin-cli importprivkey "WIF_KEY_HERE"

# Import with label and rescan
bitcoin-cli importprivkey "WIF_KEY_HERE" "label" true

# Import multiple keys
bitcoin-cli importmulti '[{"scriptPubKey":{"address":"ADDRESS"},"keys":["WIF_KEY"],"timestamp":"now"}]'
```

### Using Python (bitcoin library)

```python
from bitcoin import *

# WIF to address
priv_key_wif = "L2yeAsCPNn6KH8c..."
priv_key_hex = wif_to_key(priv_key_wif)
pub_key = privkey_to_pubkey(priv_key_hex)
address = pubkey_to_address(pub_key)

print(f"Address: {address}")
```

## üéØ Address Type Mapping

`bitcoin-keyforge` generates multiple address types. Import to wallets supporting your preferred type:

| Address Type | Format | Wallet Support |
|--------------|--------|----------------|
| **P2PKH (Legacy)** | `1...` | ‚úÖ All wallets |
| **P2SH-P2WPKH (Nested SegWit)** | `3...` | ‚úÖ Most modern wallets |
| **P2WPKH (Native SegWit)** | `bc1q...` | ‚úÖ Modern wallets |
| **P2TR (Taproot)** | `bc1p...` | ‚ö° Newer wallets only |

**Always test with small amounts first:**

1. **Import private key** into chosen wallet
2. **Verify address matches** btc-split-key output
3. **Send small test amount** (0.0001 BTC)
4. **Confirm receipt** and balance display
5. **Test sending** from imported wallet
6. **Only then import production keys**

## üö® Common Issues & Solutions

### Issue: "Invalid Private Key"
**Solution**: Ensure you're using WIF format, not hex

### Issue: "Address Mismatch"
**Solution**: Check if wallet uses different address type (P2PKH vs P2WPKH)

### Issue: "Zero Balance"
**Solution**:
- Rescan blockchain
- Check if wallet synced
- Verify correct network (mainnet vs testnet)

### Issue: "Import Failed"
**Solution**:
- Check wallet supports private key import
- Try different address derivation
- Use desktop wallet instead of mobile

## üîê Security Best Practices

### For Production Use:

1. **Air-gapped machine** - Never import on internet-connected device
2. **Hardware wallet** - Transfer to hardware wallet after import
3. **Multi-signature** - Use 2-of-3 multisig for large amounts
4. **Cold storage** - Keep majority in cold storage
5. **Regular backups** - Multiple secure backup locations

### For Testing:

1. **Testnet first** - Always test on testnet
2. **Small amounts** - Start with minimal values
3. **Known wallets** - Use reputable, well-tested wallets
4. **Verify addresses** - Double-check all addresses
